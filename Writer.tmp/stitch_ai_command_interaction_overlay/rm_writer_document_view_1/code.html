Authentication Service Architecture

Last edited 2 mins ago by Sarah
auto_awesome AI Context Active

This document outlines the architectural decisions and implementation details for the new authentication microservice. The goal is to provide a secure, scalable, and standards-compliant identity provider for the RM ecosystem.

hub Core Components

  • Identity Provider (IdP): Handles user login, MFA, and session management using OAuth2 / OIDC protocols.
  • Token Service: Issues JWT access and refresh tokens with specific scopes.
  • User Directory: Encrypted storage for user profiles and credentials using PostgreSQL.

System Flow

The authentication flow follows the standard Authorization Code Flow with PKCE for public clients (SPA/Mobile) and Client Credentials Flow for service-to-service communication.

auth-config.ts Typescript 
export const authConfig = {
  domain: process.env.AUTH_DOMAIN,
  clientId: process.env.AUTH_CLIENT_ID,
  audience: "https://api.rmwriter.com",
  scope: "read:documents write:documents"
};

Security Considerations

All tokens must be signed using RS256. The public key will be exposed via a JWKS endpoint for services to validate signatures. Sensitive environment variables are stored in Vault.
